# 開始手動測試 - 員工登入畫面

## 服務狀態

C# .NET 7.0 後端服務應該已經在運行。

## 測試步驟

### 1. 打開瀏覽器

訪問以下網址：

```
http://localhost:5000/employee
```

### 2. 員工登入畫面測試

您應該看到：
- ✅ 標題「工廠員工訂餐系統」
- ✅ 員工編號輸入框（預設值：93800）
- ✅ 密碼輸入框（預設值：1234）
- ✅ 「登入」按鈕

### 3. 執行登入測試

**測試用例 1：正確的帳號密碼**

1. 確保員工編號為：`93800`
2. 確保密碼為：`1234`
3. 點擊「登入」按鈕
4. **預期結果：**
   - ✅ 登入成功
   - ✅ 顯示「歡迎您，林淑鈺！」
   - ✅ 顯示「今日訂餐」頁籤
   - ✅ 顯示午餐和晚餐的訂購選項

**測試用例 2：錯誤的密碼**

1. 員工編號：`93800`
2. 密碼：`wrong`
3. 點擊「登入」
4. **預期結果：**
   - ❌ 顯示錯誤訊息「員工編號或密碼錯誤」

**測試用例 3：不存在的員工編號**

1. 員工編號：`99999`
2. 密碼：`1234`
3. 點擊「登入」
4. **預期結果：**
   - ❌ 顯示錯誤訊息「員工編號或密碼錯誤」

### 4. 登入成功後的畫面

登入成功後應該看到：

- ✅ 「歡迎您，[員工姓名]！」訊息
- ✅ 「登出」按鈕
- ✅ 「今日訂餐」頁籤（已選中）
- ✅ 「歷史紀錄查詢」頁籤
- ✅ 午餐訂購區域（顯示狀態、訂購按鈕）
- ✅ 晚餐訂購區域（顯示狀態、訂購按鈕）
- ✅ 截止時間提醒（午餐 08:30 / 晚餐 16:00）
- ✅ 當前時間顯示

## 常見問題排查

### 問題 1：無法訪問頁面

**解決方案：**
- 確認服務是否運行：檢查終端是否有錯誤訊息
- 確認 URL 正確：`http://localhost:5000/employee`
- 嘗試訪問 `http://localhost:5000/` 查看服務是否運行

### 問題 2：登入按鈕沒有反應

**解決方案：**
- 打開瀏覽器開發者工具（F12）
- 查看 Console 是否有錯誤訊息
- 查看 Network 標籤，確認 API 請求是否發送

### 問題 3：登入失敗但沒有錯誤訊息

**解決方案：**
- 檢查瀏覽器 Console 的錯誤訊息
- 確認 API 是否正常運行（訪問 `http://localhost:5000/swagger`）

### 問題 4：登入成功但畫面沒有更新

**解決方案：**
- 檢查瀏覽器 Console 是否有 JavaScript 錯誤
- 確認 API 響應是否正確（在 Network 標籤查看）

## 下一步測試

登入成功後，繼續測試：

1. **今日訂單查詢** - 確認能否正確顯示當前訂單狀態
2. **訂購功能** - 測試創建午餐/晚餐訂單
3. **取消訂單** - 測試取消已創建的訂單
4. **歷史查詢** - 測試查詢歷史訂單
